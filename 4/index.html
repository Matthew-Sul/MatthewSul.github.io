Part 2: Homographies

To warp images to one another, need to compute the homography matrix, 3x3 with 8 DOF. Want to map the points p from the input to p’ from the output.

Once we have our equation in the form of Ax = b, can use least squares to solve for it. Our x is the column with a to h, manually construct the 3x3 matrix in the top left and that’s the Homography. Final call would be H = computeH(image1_pts, image2_pts)

Part 3+4: Warp Images + Rectification

Created a function using recently gotten H to warp [warpImage(im,H)]. Given the homography, this allows for warping an image however I want given I provide the proper H.

Would start of getting H inverse, finding the size of the warped image by getting (x,y,1) coordinates of the src_img corners. To get destination, do H @ corners (corners being 3x4 shape) and normalize by dividing (wx,wy,w) by w to get (x,y,1). Grab the minimum and maximum x and y coors to get bounding box of the final image. Once the (x,y,1) coords gotten, do a similar step to proj3 by doing

H@destination_pts, normalize again (this time normally vs by w) and used scipy.interpolate.griddata to grab color vals of pixels to yeet on the final image.

Part B - FEATURE MATCHING for AUTOSTITCHING

Used the same images in both parts A and B.

Part 1: Detecting Corner Features in an Image

Use the given Harris Interest Point Detector (+ dist2 function) to do corner detection on images. Follow the process and ideas of the “Multi-Image Matching using Multi-Scale Oriented Patches” by Brown et al paper. After getting all of the Harris corners (the images bum-rushed with red), use Adaptive Non-Maximal Suppression (ANMS) to leave us with the points that stand out the most. We would then use these selected points to do feature descripting, matching and later on RANSAC.

Part 2 + 3: Extracting Feature Descriptor for each feature point + Matching these feature descriptors between two images

Create a 40x40 area around each point to get patches from the previous part and resize to an 8x8 area using cv2.resize. Normalized and grayscaled the image (per OH and peer rec from paper) and flatten the descriptor so now it’s 1x64. After extracting the features, want to match them together so can get a finalized point set to use for RANSAC. Compute Nearest Neighbors and construct pairs of corresponding points between the images to use for RANSAC. Can do NN both ways to ensure that the match works both ways and is more likely to be accurate.

Part 4: RANSAC (to compute Homography)

This is a robust way to get get homographies. Feature matching gets a stronger set of points but want to filter out any possible outliers to leave us with most ideal and strongest points to get the Homography (done by RANSAC). Sample 4 points randomly from the feature matching output to then get the Homography and check the amount of inliers. We repeat the process and keep the homography that has the most inliers to use again for Mosaic creation.
